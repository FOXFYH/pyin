# 挑战系统 — 完整项目说明

## 一、项目概览

1. 项目名称：挑战系统（课堂答题竞赛系统）
2. 技术栈：纯前端（HTML + CSS + JavaScript），无后端依赖，数据存储于浏览器 localStorage
3. 云同步：通过嵌入 iframe（sync.html）与云端数据库通信，支持多设备数据同步
4. 文件结构：
   - index.html — 主页面结构
   - app.js — 核心业务逻辑（约2100行）
   - style.css — 全局样式
   - mode-rules.js — 模式规则定义
   - demo-data.js — 默认学生与题库数据
   - sync.html — 云同步代理页面


## 二、页面导航与视图

1. 导航栏（顶部固定）
   - 🚀 Let's go — 首页（模式选择与答题）
   - 📊 学情 — 学生数据总览
   - 📚 题库 — 题库管理
   - 🏆 榜单 — 积分排行榜
   - ⚙️ 设置 — 系统设置
   - 考试中额外显示：模式标签 + ⏹ 结束按钮
   - 右侧：音效开关、缩放控制、全屏切换

2. 五大视图
   - view-home：模式选择 → 参赛配置 → 抽签动画 → 答题界面
   - view-students：学生学情表格
   - view-questions：题库卡片列表
   - view-leaderboard：积分排行榜（领奖台 + 底部列表）
   - view-settings：七个子面板（等级/挑战参数/AI/音效/学生/数据管理/云同步）


## 三、数据存储体系

### 3.1 本地存储（localStorage）

所有数据以 `exam_` 为前缀存入 localStorage，包括：

| 键名 | 内容 | 说明 |
|------|------|------|
| exam_students | 学生数组 | 见3.2 |
| exam_banks | 题库数组 | 见3.3 |
| exam_records | 场次记录数组 | 见3.4 |
| exam_settings | 全局设置 | 见3.5 |
| exam_examProgress | 考试进度 | 见3.6 |
| exam_wrongSets | 错题集合 | 见3.7 |
| exam_dataVersion | 数据版本号 | 当前版本5，控制默认数据初始化 |
| exam_sync_* | 云同步状态 | 上次同步时间等，不参与数据重置 |

### 3.2 学生数据结构（exam_students）

数组，每个元素：

```
{
  id: "唯一ID（时间戳+随机数）",
  name: "姓名",
  avatar: "头像（Base64图片或空字符串）",
  gender: "性别（男/女/空）",
  group: "小组名（空字符串表示未分组）"
}
```

- 写入时机：添加/编辑/导入/删除学生时
- 云同步：变更后自动通知云端

### 3.3 题库数据结构（exam_banks）

数组，每个元素：

```
{
  id: "唯一ID",
  name: "题库名称",
  description: "题库描述",
  questions: [
    {
      id: "唯一ID",
      text: "题目内容",
      options: { A: "选项A", B: "选项B", C: "选项C", D: "选项D" },
      answer: "正确答案（A/B/C/D）",
      points: 分值（默认10）
    }
  ],
  createdAt: 创建时间戳,
  _newFile: 是否新文件标记（可选）
}
```

- 写入时机：新建/编辑/导入/删除题库时
- 云同步：变更后自动通知云端

### 3.4 场次记录数据结构（exam_records）

数组，每个元素：

```
{
  id: "唯一ID",
  date: 场次时间戳,
  mode: "模式（farm/challenge/pk/practice）",
  participationType: "参赛方式（personal/group）",
  bankNames: [题库名称数组],
  timeLimit: 超时设置,
  totalQuestions: 总题数,
  totalCorrect: 答对题数,
  totalEarned: 本场总积分（含所有额外加分）,
  studentResults: {
    "学生ID": {
      id: "学生ID",
      name: "姓名",
      avatar: "头像",
      group: "小组",
      correctCount: 答对题数,
      totalCount: 总答题数,
      pointsEarned: 获得积分
    }
  },
  playerScores: { "学生ID": 积分 },
  groupScores: { "小组名": 积分 },
  questionDetails: [
    {
      index: 题号,
      text: "题目内容",
      answer: "正确答案",
      studentId: "答题学生ID",
      studentName: "答题学生姓名",
      selectedOption: "所选选项",
      correct: 是否正确,
      pointsEarned: 本题得分
    }
  ],
  results: [原始答题记录数组],
  practicePassed: 是否通过（仅练习模式）
}
```

- 写入时机：每次考试结算时追加一条
- 练习模式：仅全对结算后才写入，有错题时也写入但积分记为0
- 云同步：变更后自动通知云端
- 清空方式：设置 → 数据管理 → 清空场次数据

### 3.5 全局设置数据结构（exam_settings）

```
{
  // 等级配置
  levels: [
    { name: "等级名", minPoints: 最低积分, comment: "评语" }
  ],

  // AI配置
  aiApiUrl: "API地址",
  aiApiKey: "API密钥",
  aiModel: "模型名",

  // 音效
  soundEnabled: 是否开启,
  soundVolume: 音量（0-100）,

  // 挑战参数
  shuffleOptions: 是否打乱选项（默认true）,
  baseTimeout: 基础超时秒数（默认30，0=不限时）,
  charTimeoutCompensation: 每字超时补偿秒数（默认0.2）,
  autoNext: 是否自动下一题（默认true）,
  autoNextDelay: 自动下一题间隔秒数（默认3）,
  drawAnimation: 是否开启抽签动画（默认true）,
  drawDuration: 抽签动画时长秒数（默认3）,

  // 字号设置
  questionFontSize: 题目字号（默认30）,
  studentNameFontSize: 学生名字字号（默认35）,
  studentInfoFontSize: 学生信息字号（默认20）,
  warReportFontSize: 战报字号（默认20）,

  // 上次比赛设置（记忆功能）
  lastMode: "上次模式",
  lastParticipationType: "上次参赛方式",
  lastPlayerOrder: "上次出场顺序",
  lastQuestionOrder: "上次出题方式",
  lastAvgQuestions: 上次每生平均答题数,
  lastGroupRotation: "上次小组轮换方式",
  _lastBankIds: [上次选择的题库ID数组]
}
```

- 写入时机：保存设置时 / 开始考试时（记忆上次设置）
- 云同步：仅等级配置同步

### 3.6 考试进度数据结构（exam_examProgress）

中途退出考试时保存，用于下次恢复：

```
{
  mode: "模式",
  participationType: "参赛方式",
  questions: [题目数组],
  timeLimit: 超时设置,
  autoNext: 是否自动下一题,
  students: [学生数组],
  currentIndex: 当前题号,
  results: [已答结果数组],
  playerScores: { "学生ID": 积分 },
  totalEarned: 已获总积分,
  playerQueue: [选手队列],
  queueIndex: 队列指针,
  avgQuestions: 每生平均答题数,
  groupNames: [小组名数组],
  groupRotation: "轮换方式",
  groupMap: 小组映射,
  groupScores: { "小组名": 积分 },
  currentStudentId: "当前答题学生ID",
  _pointsPerQ: 每题分值（练习模式专用）,
  _isRetry: 是否补考（练习模式专用）,
  _bankIds: [题库ID数组],
  savedAt: 保存时间戳
}
```

- 恢复机制（按模式区分）：
  - 非练习模式（打野/挑战/PK）：页面加载或回到首页时自动恢复
  - 练习模式：不自动恢复，当用户选择同一题库+同一学生并点击开始时，弹窗询问是否继续
- 清除时机：考试结算完成后自动清除

### 3.7 错题集数据结构（exam_wrongSets）

以对象形式存储，键为 `学生ID_题库ID`：

```
{
  "学生ID_题库ID": {
    wrongQuestionIds: [错题ID数组],
    lastAttempt: 上次作答时间戳,
    bankId: "题库ID",
    pendingPoints: 累计未结算积分,
    nextPointsPerQ: 下次补考每题分值,
    lastInitialPPQ: 本轮挑战初始分值
  }
}
```

各字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| wrongQuestionIds | 数组 | 本轮答错的题目ID列表，补考时从中出题 |
| lastAttempt | 数字 | 上次作答时间，用于排序和统计 |
| bankId | 字符串 | 所属题库ID |
| pendingPoints | 数字 | 累计未结算积分，补考全对时一次性结算 |
| nextPointsPerQ | 数字 | 下次补考每题分值（递减后），首轮10→补考1为5→补考2为2→补考3为1→之后固定1 |
| lastInitialPPQ | 数字 | 本轮挑战的初始每题分值，新一轮挑战时减半使用 |

生命周期：
- 创建：首次练习某题库有错题时
- 更新：每次补考结算时（更新错题集、累积未结算积分、计算下次分值）
- 清空：补考全对时（wrongQuestionIds清空，pendingPoints归零，保留lastInitialPPQ供新一轮使用）
- 不上传云端：仅保存在本地

### 3.8 云端同步

1. 同步方式：通过 iframe（sync.html）与云端数据库通信，使用 postMessage 双向传递数据
2. 同步数据类型及方向：

| 数据类型 | 同步方向 | 说明 |
|----------|----------|------|
| students | 双向 | 学生数据变更后自动推送，也接收云端更新 |
| banks | 双向 | 题库数据变更后自动推送，也接收云端更新 |
| records | 双向 | 场次记录变更后自动推送，也接收云端更新 |
| levels | 双向 | 等级配置变更后自动推送，也接收云端更新 |

3. 同步时机：
   - 数据变更时自动通知（防抖延迟可配置，默认3秒）
   - 手动点击同步按钮
   - 页面重新可见时自动同步（visibilitychange事件）
4. 同步配置项：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| enabled | true | 是否开启云同步 |
| webhookUrl | - | Webhook地址 |
| token | - | 认证令牌 |
| cloudCheckInterval | 5秒 | 云端检查间隔 |
| syncDeltaInterval | 3秒 | 同步防抖延迟 |
| retryCount | 5 | 重试次数 |
| retryInterval | 2秒 | 重试间隔 |

5. 冲突处理：云端数据更新时，直接覆盖本地对应数据，并刷新页面显示


## 四、五种比赛模式

### 4.1 🌾 打野模式

1. 分值规则：答对一题得10分，答错不得分，每题分值固定
2. 参与方式：个人或小组均可
3. 出场顺序：可选按顺序、公平随机、真随机
4. 结束条件：每生平均答题数达标
5. 小组额外规则：获胜小组全员每题额外 +5 分

### 4.2 🎯 挑战模式

1. 分值规则：答对一题得5分，答错不得分
2. 正确率补偿（挑战结束后结算）：
   - 100% → 每题额外 +15 分
   - ≥95% → 每题额外 +12 分
   - ≥90% → 每题额外 +9 分
   - ≥85% → 每题额外 +6 分
   - <85% → 无额外补偿
3. 参与方式：个人或小组均可
4. 小组额外规则：获胜小组全员每题额外 +5 分

### 4.3 ⚔️ PK模式

1. 分值规则：答对一题得5分，获胜者每题额外 +10 分
2. 参与限制：2-8名选手
3. 参与方式：个人或小组均可
4. 小组额外规则：获胜小组全员每题额外 +5 分

### 4.4 📝 练习模式（挑战题库模式）

#### 一、基础设定

1. 参与限制：单次仅一名学生参与，需作答单个题库内全部题目
2. 初始分值：首轮挑战每题基础分为10分
3. 数据存储分类：分为本地临时记录、云端共用数据库两类
4. 场次定义：完成一次分数结算，即为一个独立场次；重复挑战、分值递减的新一轮挑战，均单独计为新场次，云端数据库每条记录对应一个场次

#### 二、首轮考试流程

1. 作答要求：必须完整答完全部题目，中途退出可接续进度，不可跳题。完成全题作答后，进度记录生效
2. 结果判定：
   - 全答对：直接结算分数，本轮挑战结束
   - 存在错题：统计答对题目得分，生成未结算积分，同时记录错题编号，进入补考环节
3. 本地临时记录规则：
   - 学生首次挑战某题库，新建本地记录，实时保存答题进度
   - 完成题库全题作答后，停止进度记录，标记该生已完成此题库一轮首轮考试，永久留存本轮错题编号
   - 该类进度、错题数据仅保存在本地，不上传云端

#### 三、补考规则（可多次循环）

1. 出题规则：每轮补考试题 = 当前遗留错题 + 随机抽取同等数量的历史答对题目。补考失利则更新错题集，下一轮补考沿用新错题集，组题规则不变
2. 分值规则：
   - 首次补考：题目分值在上一档基础上减半，结果取整数
   - 后续每轮补考：分值持续减半，最低固定为1分，不再降低
   - 得分累加：每轮补考得分，实时并入前期未结算积分
3. 结果判定：
   - 仍有错题：生成新错题集，继续下一轮补考
   - 全部答对：挑战通关，统一结算所有累计未结算积分，本轮挑战正式结束

#### 四、数据上传与场次记录

1. 上传时机：仅当本轮挑战完成最终分数结算后，才将本场数据同步至云端共用数据库
2. 场次统计：每完成一次结算，算作一个独立场次，云端数据库单独生成一条对应记录

#### 五、新一轮重复挑战规则

1. 分值调整：再次发起同一题库挑战时，本轮初始分值在上一轮挑战初始分值基础上减半
2. 流程复用：首轮考试、出题规则、补考机制、分值递减规则、本地记录规则全部沿用上述要求
3. 场次归属：新一轮挑战完成结算后，单独计为新场次，云端数据库新增一条记录

#### 六、分值递减示例

```
首轮挑战（初始10分）：
  首轮考试 → 每题10分
  补考1 → 每题5分（10÷2）
  补考2 → 每题2分（5÷2取整）
  补考3 → 每题1分（2÷2）
  补考4+ → 每题1分（最低1分）

新一轮挑战（初始5分 = 上一轮初始10分÷2）：
  首轮考试 → 每题5分
  补考1 → 每题2分
  补考2 → 每题1分
  补考3+ → 每题1分

再新一轮挑战（初始2分 = 上一轮初始5分÷2）：
  首轮考试 → 每题2分
  补考1 → 每题1分
  补考2+ → 每题1分
```

### 4.5 🧠 智能抽查模式

#### 一、基础设定

1. 参与方式：默认全员参与，个人或小组均可
2. 出场顺序：智能随机（根据正确率排名加权抽取，不可切换）
3. 智能强度参数：默认5，可在设置中调整（范围1-20）

#### 二、概率点数计算

1. 每人基础概率点数为100
2. 概率点数公式：100 +（正确率排名 - 1）× 智能强度
3. 正确率排名规则：按历史正确率从高到低排列，正确率相同则答题数多者优先，无答题记录者排最末
4. 排名越低（正确率越差），概率点数越高，越容易被抽中
5. 每完成一场才重新计算正确率排名

#### 三、积分修正规则

1. 基础分值：10分
2. 实际分值公式：⌊基础分 / 概率点数 × 100⌋（取整，最低1分）
3. 示例（智能强度5，30人班级）：
   - 第1名（概率点数100）：分值 = ⌊10/100×100⌋ = 10分
   - 第15名（概率点数170）：分值 = ⌊10/170×100⌋ = 5分
   - 第30名（概率点数245）：分值 = ⌊10/245×100⌋ = 4分
4. 设计目的：正确率低的学生更容易被抽中，但每题分值较低；正确率高的学生不易被抽中，但答对可得满分

#### 四、智能强度设置

1. 设置位置：选择智能抽查模式后，配置区显示"智能强度"滑块
2. 实时预览：滑块旁显示当前末名概率点数和班级人数
3. 示例：强度5、30人 → 末名概率点数 = 100 + 29×5 = 245
4. 强度越大，首末名概率差距越大


## 五、考试流程详解

### 5.1 模式选择与配置

1. 进入首页 → 选择模式（打野/挑战/PK/练习/智能抽查）
2. 系统自动恢复上次比赛设置（参赛方式、出场顺序、出题方式等）
3. 配置项：
   - 参赛方式：👤 个人 / 👥 小组
   - 题库选择：弹窗选择，四列展示，支持多选
   - 出场顺序：按顺序出场 / 公平随机（≥3人）/ 真随机（≥2人）
   - 出题方式：公平随机 / 真随机
   - 每生平均答题数
   - 超时设置：基础秒数 + 每字补偿
   - 小组轮换方式：交替 / 顺序（仅小组模式）
4. 人员选择：
   - 个人模式：点选学生，全选/取消全选
   - 小组模式：勾选小组，自动选中组内学生
   - 练习模式：仅限选择1人
   - PK模式：2-8人

### 5.2 抽签动画

1. 触发条件：出场顺序为公平随机或真随机，且参赛人数 > 1，且抽签动画开关开启
2. 流程：随机滚动显示人名 → 持续可配置时长 → 选中队列中的下一位选手
3. 单人模式自动跳过动画
4. 按顺序出场模式不触发动画
5. 可随时点击停止按钮提前结束

### 5.3 答题流程

1. 显示当前选手信息（头像、姓名、小组、等级、积分）
2. 显示题目和选项（选项顺序默认打乱）
3. 超时计时器：基础秒数 + 题目字数 × 每字补偿
4. 选择选项后：
   - 答对：显示正确反馈 + 得分弹出，自动/手动下一题
   - 答错：显示错误反馈 + 正确答案，自动/手动下一题
   - 超时：视为答错
5. 撤销功能：答题后可点击"↩️ 撤销重选"回退本次选择（重新计时）

## 十二、关键算法

### 12.1 公平随机出题

1. 将所有题目打乱顺序
2. 按顺序分配，确保每题至少出一次
3. 题目不够时循环套用

### 12.2 公平随机出场

1. 将所有选手打乱顺序生成队列
2. 按队列顺序轮流答题
3. 队列用完后重新打乱生成新队列
4. 确保每人出场次数基本均等

### 12.3 超时计算

```
每题超时 = 基础超时(秒) + 题目字数 × 每字补偿(秒/字)
```

### 12.4 练习模式积分计算

```
首轮每题分值 = _getInitialPointsPerQ() → 默认10，新一轮 = 上一轮初始÷2
补考每题分值 = Math.max(1, Math.floor(上次分值 / 2))
答对得分 = 每题分值
未结算积分 = 累计所有轮次答对得分
全对结算 = 所有未结算积分一次性入账
```

### 12.5 挑战模式积分补偿

```
正确率 = 答对数 / 总题数
100% → 每题 +15
≥95% → 每题 +12
≥90% → 每题 +9
≥85% → 每题 +6
<85% → 无补偿
最终积分 = 基础得分 + 补偿得分
```

```
用户操作
  │
  ├─ 模式选择 → App.Exam.enterMode()
  │     │
  │     ├─ 恢复上次设置（App.Storage.getSettings）
  │     └─ 渲染配置界面
  │
  ├─ 开始考试 → App.Exam.startExam()
  │     │
  │     ├─ 构建题目池（公平随机/真随机）
  │     ├─ 构建选手队列（按顺序/公平随机/真随机）
  │     ├─ 保存当前设置到本地存储
  │     └─ 进入答题循环
  │
  ├─ 答题 → App.Exam.selectOption()
  │     │
  │     ├─ 计算得分
  │     ├─ 更新战报
  │     ├─ 保存进度（_saveProgress）
  │     └─ 自动/手动下一题
  │
  ├─ 结束考试 → App.Exam.showResult()
  │     │
  │     ├─ 练习模式：计算未结算积分 / 补考分值
  │     ├─ 其他模式：按规则计算积分
  │     ├─ 生成本地记录（App.Storage.setRecords）
  │     ├─ 全对/结算 → 云同步（App.Sync.syncNow）
  │     └─ 显示结果页面
  │
  └─ 中途退出 → App.Exam._doEndExam()
        │
        ├─ 练习模式：保存进度，提示可继续
        └─ 其他模式：保存进度，退出
```
